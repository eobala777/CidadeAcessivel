2. Visão Geral
O Cidade Acessível é um sistema voltado à sustentabilidade urbana e inclusão social. Através de um aplicativo móvel, os cidadãos podem relatar falhas na infraestrutura pública, permitindo uma gestão urbana mais participativa e focada na mobilidade universal. [1] 

* Objetivo: Criar um ecossistema sustentável para reporte de vias danificadas, obstáculos à acessibilidade, falhas em iluminação e problemas em transportes públicos.
* ODS Relacionada: ODS 11 – Cidades e Comunidades Sustentáveis.
* Justificativa: Promover ambientes adaptados e elevar a qualidade de vida urbana por meio da tecnologia e do engajamento social. [2] 

3. Escopo do Aplicativo## 3.1 Funcionalidades Principais

* Reporte em Tempo Real: Registro imediato de problemas urbanos.
* Foco em Acessibilidade: Prioridade para demandas de Pessoas com Deficiência (PcD).
* Validação Comunitária: Sistema de participação coletiva para confirmar a veracidade dos incidentes relatados.
* Sistema utilizando o Google Maps: Usuarios podem indicar as coordenadas do exato local onde o problem está acontecendo

3.2 Diferenciais
Diferente de soluções de zeladoria genéricas, o projeto foca especificamente na mobilidade para PcD e no impacto direto das falhas estruturais na autonomia dos cidadãos.
3.3 Público-Alvo

* Cidadãos em geral, com ênfase em usuários que necessitam de infraestrutura acessível.

4. Documentação Técnica Inicial## 4.1 Requisitos Funcionais (RF)

| ID | Descrição |
|---|---|
| RF01 | Cadastro e autenticação de usuários (Login). |
| RF02 | Registro de ocorrências com localização detalhada (Estado, Cidade, Rua, Bairro e Referência). |
| RF03 | Categorização de problemas (ex: calçada quebrada, semáforo, bueiro, iluminação). |
| RF04 | Descrição detalhada do incidente. |
| RF05 | Sistema de apoio/confirmação de problemas por outros usuários. |
| RF06 | Canal de exportação de dados para órgãos responsáveis (Prefeituras/Secretarias). |
| RF07 | Dashboard ou lista de acompanhamento de problemas relatados. |

4.2 Requisitos Não Funcionais (RNF)

* Segurança: Proteção dos dados dos usuários conforme a LGPD.
* Disponibilidade: O sistema deve estar operacional 24/7.
* Compatibilidade: Funcionamento em diversas versões de sistemas operacionais móveis.
* Manutenibilidade: Código estruturado para facilitar atualizações futuras.
* Confiabilidade: Garantia de que as informações reportadas cheguem ao destino correto.
* Rapidez: Baixa latência no carregamento de mapas e envio de dados.
